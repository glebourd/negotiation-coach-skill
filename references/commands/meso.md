# meso [negotiation] — Build MESO Packages

## Purpose
Build MESO (Multiple Equivalent Simultaneous Offers) packages for a negotiation. MESOs reveal counterpart priorities without requiring direct disclosure, signal flexibility and cooperation, and create more opportunities to find value-creating trades.

## When MESOs Are Most Valuable
- Multi-issue negotiations where priorities differ across issues
- After an impasse or adversarial prior exchange (rebuilds trust signals without appearing to capitulate)
- When direct questioning has hit resistance — the counterpart won't tell you what they care about
- Async / email negotiation — gives both sides time to evaluate options properly
- When you need to demonstrate flexibility without conceding on your core interests
- When you want to gather information about counterpart priorities without asking directly

## Prerequisite
A scoring system is required to build valid MESOs. If one doesn't exist, run `score` first, then return here.

## Workflow

### Step 1: Confirm Scoring System
Load the scoring system for this negotiation. All packages must score within ~10% of each other for you — otherwise they're not truly equivalent and you'd prefer one over the others.

### Step 2: Generate Package Options
Build 3 packages. Each package must:
- Score roughly equally for you (within ~10% of each other)
- Differ meaningfully in composition — not just minor variations on one issue
- Be genuinely acceptable (above your RP, ideally near aspiration)

For each package, assign values across all issues and calculate the total score.

### Step 3: Verify Equivalence
Show the scoring breakdown for each package. If they're not truly equivalent, adjust. The equivalence is what makes MESOs work — if you prefer one significantly, that preference will show and undermine the strategy.

### Step 4: Frame for Presentation
Write the MESO presentation in the user's voice:
- Lead with the framing: "I've put together a few options that all work for me — I'm genuinely flexible between them."
- Present packages clearly (Option A / Option B / Option C) with each issue stated plainly
- End with an invitation: "Which of these is closest to what works for you? Or is there a combination we should explore?"

The framing matters: this signals flexibility and cooperation, not weakness or desperation.

### Step 5: Prepare for Responses

**If counterpart picks one:** That choice reveals their priorities. Note what it tells you about their issue weights — this is intelligence you couldn't get by asking directly.

**If counterpart rejects all three:** Ask what's missing rather than immediately conceding. "What would need to change for one of these to work?" — this surfaces their actual interests without a unilateral concession.

**If counterpart modifies one:** That's an invitation to negotiate, not a concession demand. Treat it as information and respond with "If you need [their change], I'd need [something in return]."

**If counterpart accepts immediately:** Consider whether you anchored too low. The appropriate response to an immediate acceptance is typically to add something small before confirming ("I'd want to add [minor term] to make that work fully") — not because you're greedy, but because an instant acceptance often means you left value on the table.

## Output Format
- Scoring table showing all three packages with issue-by-issue breakdown and total scores
- Presentation language in the user's voice
- Interpretation guide: what different counterpart responses tell you

## State Update
Note MESO packages prepared in the Active Negotiations entry. After the actual negotiation, record which package they chose (or rejected) and what it revealed about their priorities.
